Welcome to the new BETA Quick-Build page for Goma 6.0! We have made a run script that will allow you to easily download all of the necesarry supporting files for Goma 6.0, and we have also included simple instructions on how to correctly configure Goma 6.0 so that it is ready for immediate use!

Download build script

Download the build script here

Untar the build script, go into the directory, and move the build script to where you want the Goma Libraries to be made.

Once the build script is in the desired location, simply run the build script by entering

buildLibScript.sh in the command prompt.

As soon as that finishes downloading, you have the essential libraries inorder to run Goma 6.0!

Build Goma

  • GOMA:
    • Go to the GitHub page goma/goma.
    • Copy the HTTPS clone URL on the right hand side of the page, then cd into where you want your goma directory to be and run git clone

    cd into your new goma directory and change the settings.mk-example into your settings.mk file by running cp settings.mk-example settings.mk

    Edit the variables GOMA_LIBS, SEACAS_TOP, and PREFIX in the settings.mk file

    In the settings.mk, add

  • FRONT_LIB =
    • and
  • UMFPACK_TOP = $(GOMA_LIBS)/UMFPACK-5.4/UMFPACK
    • after the PREFIX.

    After making sure that the paths are set correctly, run:

  • make
    • 
	  
  • make install

    • I did not run into any issues with building goma. Just make sure all the paths are pointed to the correct compilers and the libraries.

    Build brk/fix

  • brk/fix:

    • Go to the GitHub page goma/brkfix.
    • Copy the HTTPS clone URL on the right hand side of the page, then cd into your Goma directory and run git clone

    Check the following macros before doing a make.

    		CC
		ACCESS
		COPTIMIZEFLAGS

    Brk/fix compiled without any major issues.
